# Stratim — Обзор стандарта

## Что такое Stratim?

Stratim — современный текстовый формат для создания и структурирования электронных книг. Он разработан как прямая альтернатива FictionBook 2 (FB2) и Markdown, устраняя их фундаментальные ограничения:

### Проблемы, которые решает Stratim

| Формат | Проблема | Решение Stratim |
|--------|----------|-----------------|
| **FictionBook 2** | XML-многословие, сложность ручного редактирования | Текстовый синтаксис с командами `~` |
| **Markdown** | Неоднозначность, эвристики парсинга | Детерминированные последовательные дескрипторы |
| **Оба** | Сложная вложенность, неявные правила | Явный стек с модификаторами `^`/`v` |

## Философия формата

### 1. Последовательность вместо вложенности
Документ читается линейно. Каждая новая команда `~` неявно закрывает предыдущий контекст.

```stratim
~ # (ch1) [Глава 1]
Текст главы
~ # (ch2) [Глава 2]  ← ch1 автоматически закрыт
```

### 2. Нативные метаданные
Никаких JSON или YAML. Все поля описываются через `~` команды.

```stratim
~ t [Название книги]
~ a [Фамилия] first="Имя"
~ l ru
```

### 3. Разделитель блоков
Строго **одна пустая строка** разделяет метаданные и тело документа.

```stratim
~ t [Книга]
~ l ru

~ # (ch1) [Глава]  ← пустая строка обязательна
```

### 4. Форматирование как аномалия
Разметка `:::` разрешена только внутри контента, максимум 3 уровня вложенности.

```stratim
::: E курсив ::: и ::: S жирный :::
```

### 5. Естественный поток текста
- `\n` → новый абзац `<p>`
- Пустая строка → `<empty-line/>`
- Пробелы нормализуются (множественные → один)

## Ключевые возможности

### 📋 Метаданные и сборка блоков

```stratim
~ t [Механика Stratim]
~ a [Петров] first="Алексей"
~ x (city) [Москва]
~ x (year) [2026]

~ * (title) [Титульный лист] @a,t,city,year type="fulllist"
```

**Результат:** Автоматическая сборка центрированного титульного блока с разрывом страницы.

### 🔗 Универсальные ссылки

```stratim
См. #первую главу[ch1] и #приложение[appendix].
```

### 📝 Сноски

```stratim
Квантовая запутанность описана в #1935 году[fn_epr].

~ f (fn_epr) [ЭПР-парадокс, 1935.]
```

**Автоматика:**
- Сквозная нумерация `[1]`, `[2]`...
- Сборка в `<body name="notes">`
- Обратные ссылки

### 📊 Нативные списки

```stratim
1. Пункт уровня 0
1.1. Вложенный пункт
1.2. Ещё вложенный
2. Следующий пункт уровня 0
```

**Детерминизм:** Глубина = количество сегментов - 1. Отступы игнорируются.

### 🎨 Форматирование

| Тег | Назначение | Пример |
|-----|------------|--------|
| `D` | Буквица | `::: D П ервая :::` |
| `E` | Курсив | `::: E важно :::` |
| `S` | Жирный | `::: S ключевое :::` |
| `C` | Код | `::: C \lang d\ code :::` |
| `R` | Сырой текст | `::: R сохранить пробелы :::` |

### 🔀 Управление вложенностью

```stratim
~ # (ch1) [Глава 1]
~ # ^ (sec1) [Раздел 1.1]  ← вглубь
~ # ^ (sec1_1) [Подраздел] ← ещё вглубь
~ # v                       ← на уровень вверх
~ # (sec2) [Раздел 1.2]    ← на том же уровне
```

## Быстрый старт

### 1. Установка
```bash
git clone https://github.com/YOUR_USERNAME/stratim-standard.git
cd stratim-standard/reference-parser
dub build --build=release
```

### 2. Создание книги
Создайте `book.stratim`:
```stratim
~ t [Моя книга]
~ a [Автор] first="Имя"
~ l ru

~ * (title) [Титул] @a,t type="fulllist"

~ # (ch1) [Глава 1]
Текст главы с ::: E акцентом ::: и #ссылкой[fn1].

~ f (fn1) [Примечание.]
```

### 3. Конвертация
```bash
./stratim book.stratim book.fb2
```
