## 1. Введение

Stratim — текстовый формат описания структуры электронных книг, построенный на принципе **последовательных дескрипторов**. Формат разработан как прямая альтернатива FictionBook 2 (XML) и Markdown, устраняя их фундаментальные ограничения: многословность, эвристики парсинга и неявные правила вложенности.

### Философия формата
1. **Детерминизм:** Одна команда → один предсказуемый результат. Никаких догадок парсера.
2. **Явная структура:** Управление иерархией через стек и модификаторы `^`/`v`.
3. **Разделение ответственности:** Структура задаётся командами `~`, форматирование изолировано в дескрипторы `:::`.
4. **Естественный поток:** `\n` разделяет абзацы, пробелы нормализуются, отступы не влияют на логику.

## 2. Метаданные

Метаданные описываются исключительно командами `~` в начале файла. JSON/YAML фронтматтер отсутствует. Блок метаданных и блок тела документа разделяются **строго одной пустой строкой** (`\n\n`).

### 2.1 Синтаксис мета-команд
```text
~ <символ> (внутреннее_имя) [внешнее_имя] @массив атрибуты
```

| Символ | Назначение | FB2-эквивалент | Атрибуты / Примечания |
|--------|------------|----------------|------------------------|
| `~ i` | Идентификатор документа | `<document-info><id>` | `type="isbn\|uuid\|custom"` |
| `~ t` | Заголовок книги | `<title-info><book-title>` | `lang="ru"` (опц.) |
| `~ a` | Автор / Участник | `<title-info><author>` | `first`, `mid`, `last`, `role` |
| `~ g` | Жанр | `<title-info><genre>` | Значение из реестра FB2 2.1 |
| `~ l` | Язык текста | `<title-info><lang>` | BCP 47 / ISO 639-1 |
| `~ d` | Дата | `<title-info><date>` | `type="publish\|creation"`, `YYYY-MM-DD` |
| `~ k` | Ключевые слова | `<title-info><keywords>` | Разделитель `;` или повтор команд |
| `~ n` | Аннотация | `<title-info><annotation>` | Допускает `:::` дескрипторы |
| `~ c` | Обложка | `<title-info><coverpage>` | `src="path.jpg"`, `alt="..."` |
| `~ s` | Серия / Цикл | `<title-info><sequence>` | `name="..."`, `num="..."` |
| `~ p` | Издательские данные | `<publish-info>` | `name`, `city`, `year`, `isbn` |
| `~ u` | Исходный источник | `<document-info><src-url>` | `url`, `owner` |
| `~ x` | Кастомное поле | `<custom-info>` | Требует `(ключ)` в `^[a-zA-Z0-9_-]+$` |
| `~ 1`..`~ 9` | Быстрые слоты | `<custom-info type="N">` | Для шаблонов и каталогов |

### 2.2 Правила
- Пустые строки внутри мета-блока **запрещены**.
- Первая встретившаяся пустая строка интерпретируется **только** как разделитель.
- Обязательные поля: `~ t` и `~ l`. Отсутствие разделителя → `MISSING_META_SEPARATOR`.
- Повторяющиеся команды создают массивы в FB2. Дубликаты ключей `~ x` → `DUPLICATE_CUSTOM_KEY`.

### 2.3 Пример
```text
~ t [Механика Stratim]
~ a [Петров] first="Алексей" mid="Иванович" role="author"
~ l ru
~ d [2026-04-10] type="publish"
~ x (city) [Москва]
~ x (version) [0.1]

```
*(Обратите внимание на пустую строку после метаданных)*

## 3. Структура тела документа

Тело начинается сразу после разделителя. Заголовок книги определяется в метаданных (`~ t`). Управление иерархией осуществляется через **явный стек узлов**. Новая команда неявно закрывает предыдущий контекст.

### 3.1 Синтаксис команд тела
```text
~ <символ> [модификатор] (внутреннее_имя) [внешнее_имя] @массив атрибуты
```

| Символ | Назначение | FB2-эквивалент | Примечание |
|--------|------------|----------------|------------|
| `*` | Титульный / корневой раздел | `<section>` | Визуальная обёртка титула |
| `#` | Раздел / Глава | `<section>` | Основной структурный блок |
| `=` | Поэма / Стихотворение | `<poem>` | Листовой, `^` запрещён |
| `|` | Эпиграф | `<epigraph>` | Может идти до первой главы |
| `>` | Иллюстрация / Медиа | `<image>` | Требует `src="..."` |
| `_` | Аннотация раздела | `<annotation>` | Листовой |
| `-` | Визуальный разделитель | `<empty-line/>` | Без контента |
| `/` | Разрыв страницы | `<style>page-break...</style>` | Инструкция рендеринга |
| `f` | Определение сноски | `<body name="notes">` | Структурный узел типа `FOOTNOTE` |

**Модификаторы стека:**
- `^` — вглубь (дочерний узел)
- `v` — на уровень родителя
- *отсутствует* — тот же уровень (сосед)

**Оператор массива `@`:**
- `@ключ1,ключ2` — список ключей метаданных или анкоров.
- Контекст определяется атрибутом `type`:
  - `type="fulllist"` → сборка титульного блока из метаданных (авто-разрыв страницы).
  - `type="reflist"` → библиография из анкоров (нумерация по ГОСТ).
  - `type="notelist"` → примечания из анкоров (сборка в `<body name="notes">`).

### 3.2 Логика стека
- `~ # ^` → вложенный раздел (глубина +1)
- `~ #` → раздел на том же уровне
- `~ # v` → возврат на уровень родителя (глубина -1)
- Максимальная глубина стека: `15` уровней.

### 3.3 Пример структуры
```text
~ * (title_page) [Титульный лист] @a,t,city type="fulllist"

~ # (ch1) [Глава 1. Принципы]
Текст главы.
~ # ^ (sec1) [Раздел 1.1]
Вложенный текст.
~ # v
~ # (sec2) [Раздел 1.2]
Соседний раздел.
```

## 4. Обработка текста

### 4.1 Нормализация и абзацы
- **Пробелы:** Все последовательности пробелов/табуляций сводятся к одному пробелу `0x20`. Отступы в начале строки игнорируются парсером структуры.
- **Перевод строки `\n`:** Разделяет абзацы → `<p>...</p><p>...</p>`.
- **Пустая строка в теле:** → `<empty-line/>`.
- **Текст вне команд:** До мета-блока или после разделителя без команды → ошибка `ORPHAN_TEXT`.

### 4.2 Нативные списки
Списки определяются **лексическим префиксом**. Отступы не влияют на структуру.

**Нумерованные:**
```text
1. Пункт уровня 0
1.1. Пункт уровня 1
1.2. Пункт уровня 1
2. Пункт уровня 0 (сосед к "1.")
2.1.5. Глубокий пункт уровня 2
```
**Алгоритм глубины:** `Depth = (количество сегментов в номере) - 1`.
- `1.` → 1 сегмент → Depth 0
- `1.1.` → 2 сегмента → Depth 1
- Парсер автоматически закрывает открытые уровни при уменьшении глубины.

**Ненумерованные:**
```text
- Маркер A (всегда Depth 0 в v0.1)
- Маркер B
```

**Завершение списка:** Пустая строка, команда `~` или текст без префикса.

## 5. Дескрипторы форматирования (`:::`)

Форматирование считается **аномалией** и разрешено только внутри текстового контента.
```text
::: TAG содержимое :::
```

| Тег | Назначение | FB2-маппинг |
|-----|------------|-------------|
| `D` | Буквица | `<span class="drop-cap">` |
| `E` | Курсив | `<emphasis>` |
| `S` | Жирный | `<strong>` |
| `U` | Подчёркивание | `<underline>` |
| `K` | Зачёркивание | `<strikethrough>` |
| `C` | Код | `<code>` |
| `Q` | Inline-цитата | `<emphasis type="cite">` |
| `F` | Встроенная сноска-ссылка | `<a epub:type="noteref">` |
| `N` | Боковое примечание | `<annotation>` |
| `W` | Неразрывный перенос | `&#8209;` |
| `R` | Сырой текст | `<p xml:space="preserve">` |

**Правила:**
- **Лимит вложенности:** строго ≤3 уровней. Превышение → `NESTING_OVERFLOW`.
- Внутри `:::` отключается парсинг Markdown. Пустые строки сохраняются как `<empty-line/>`.
- **Аномальный код:** `::: C \lang <module>\ ... :::` переключает лексер в режим `RAW_CODE` (нормализация отключена, пробелы и `\n` сохраняются буквально). `<module>` валидируется по `^[a-zA-Z0-9_]+$`.

## 6. Переменные и подстановки

Оператор `(* ключ)` разрешается в текстовом потоке тела документа.
```text
Книга «(* t)». Автор: (* a). Статус: (* 1).
```

**Правила разрешения:**
1. **Ключ:** стандартный символ (`t`, `l`, `a`, `g`, `d`), цифра `1`..`9`, или ключ из `~ x`.
2. **Авто-формат:** `(* a)` автоматически форматируется как `Фамилия И.О.` (объединение всех авторов через `, `).
3. **Порядок:** Разрешение происходит **после** построения AST, **до** XML-экранирования.
4. **Рекурсия:** Запрещена. `(* (* t) )` → `RECURSIVE_VAR`.
5. **Fallback:** Ключ не найден → `WARNING: UNRESOLVED_KEY`, в вывод помещается литерал `(* ключ)`.
6. **Область:** Использование вне текстового потока (в командах `~` или `:::`) → `VAR_IN_STRUCTURE`.

## 7. Ссылки и сноски

### 7.1 Универсальный синтаксис ссылки
```text
#текст[id]
```
| Элемент | Описание |
|---------|----------|
| `#` | Маркер inline-ссылки (только в текстовом потоке) |
| `текст` | Отображаемый текст. Любой UTF-8 до символа `[`. |
| `[id]` | Целевой якорь. Должен существовать как `(внутреннее_имя)` в дереве. |

**Пример:** `См. #первую главу[ch1] и #источник[bib01].`

### 7.2 Механизм сносок
Сноска — специализированный тип ссылки. Тип определяется структурной командой создания анкора.

1. **Ссылка в тексте:** `#текст[id]` или `#[1][fn1]`
2. **Определение:** `~ f (id) [содержимое сноски с ::: форматированием :::]`
3. **Обработка парсером:**
   - Все `~ f` узлы помечаются флагом `TYPE_FOOTNOTE`.
   - При сериализации ссылки на такие узлы оборачиваются в `<sup>`.
   - Все определения автоматически собираются в `<body name="notes">`.
   - Генерируются обратные ссылки `l:href="#src_id"`.
   - Сквозная нумерация `[1]`, `[2]`... формируется по порядку появления ссылок в потоке чтения.

**Пример:**
```text
Квантовая запутанность описана в #1935 году[fn_epr].

~ f (fn_epr) [ЭПР-парадокс впервые описан в работе Эйнштейна, Подольского и Розена (1935).]
```

## 8. Валидация и соответствие

Документ считается валидным, если выполнены все условия:
1. ✅ Присутствует разделитель пустой строкой после мета-блока.
2. ✅ Все `(внутреннее_имя)` уникальны в пределах файла.
3. ✅ Глубина вложенности `:::` не превышает 3.
4. ✅ Используются только зарезервированные символы команд.
5. ✅ Стек вложенности не нарушает правила переходов `^`/`v` (макс. 15).
6. ✅ Обязательные мета-поля `~ t` и `~ l` присутствуют.
7. ✅ Нет рекурсии в операторах `(* ключ)`.
8. ✅ Все анкоры в `@` для `reflist`/`notelist` и в `#text[id]` существуют в AST.

**Коды ошибок:** `MISSING_META_SEPARATOR`, `DUPLICATE_ID`, `NESTING_OVERFLOW`, `UNKNOWN_COMMAND`, `DEPTH_OVERFLOW`, `MISSING_REQUIRED_META`, `RECURSIVE_VAR`, `BROKEN_REFERENCE`, `TYPE_MISMATCH_LINK`.

## 9. Маппинг на FictionBook 2.1

| Конструкция Stratim | FB2 2.1 эквивалент |
|---------------------|-------------------|
| `~ t [Title]` | `<title-info><book-title>Title</book-title>` |
| `~ a [Last] first="F"` | `<title-info><author><first-name>F</first-name><last-name>Last</last-name></author>` |
| `~ x (key) [val]` | `<custom-info type="x" info-type="key" info="val"/>` |
| `~ # (id) [T] @a,t type="fulllist"` | `<section id="id"><title><p>T</p></title><p align="center">Author</p><empty-line/><p align="center">Title</p></section>` + `<style>page-break-before: always</style>` |
| `~ # ^` | Вложенный `<section>` |
| `~ = (id)` | `<poem><title><p>id</p></title><stanza>...</stanza></poem>` |
| `~ | (id)` | `<epigraph><p>...</p></epigraph>` |
| `~ > src="..." alt="..."` | `<image l:href="..." description="..."/>` |
| `~ -` | `<empty-line/>` |
| `~ /` | `<style>page-break-before: always</style>` |
| `#текст[fn_id]` (где `fn_id` определён через `~ f`) | `<sup><a l:href="#fn_id" type="note">текст</a></sup>` |
| `~ f (fn_id) [Содержимое]` | `<body name="notes"><section><p><a name="fn_id"/><a l:href="#src_fn_id">[N]</a> Содержимое</p></section></body>` |
| `::: E текст :::` | `<emphasis>текст</emphasis>` |
| `::: S текст :::` | `<strong>текст</strong>` |
| `::: C \lang d\ код` | `<code class="stratim-lang-d" xml:space="preserve">код</code>` |
| `::: R текст` | `<p xml:space="preserve">текст</p>` |
| `(* a)` | Автоматическое объединение авторов `Фамилия И.О.` |

## 10. Версионирование

Формат следует **семантическому версионированию (SemVer 2.0.0)**.
- `0.1` означает начальную стабильную спецификацию.
- Обратная совместимость синтаксиса команд гарантируется в пределах мажорной версии (`1.x.x`).
- Изменения, ломающие обратную совместимость парсинга или структуры AST, выпускаются как мажорные версии (`2.0.0`).
- Патч-версии (`0.1.1`) используются для исправлений ошибок валидации, уточнений маппинга и оптимизаций парсера без изменения синтаксиса.
